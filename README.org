* udpiq-rs

Stream raw IQ samples from an ALSA capture device over UDP. Built for
Raspberry Pi based SDR transceivers.

Each ALSA period is split into MTU-sized UDP packets (1472 bytes) to
avoid IP fragmentation on standard Ethernet.

Compatible with GNU Radio UDP source (port 7373).

** Pre-built binaries

Pre-built binaries for x86-64 and aarch64 are available from GitHub
Actions. Go to the [[https://github.com/ast/udpiq-rs/actions][Actions tab]], select the latest successful build,
and download the artifact for your architecture:

- =udpiq-rs-x86-64=
- =udpiq-rs-aarch64=

#+begin_src sh
unzip udpiq-rs-aarch64.zip
chmod +x udpiq-rs
sudo mv udpiq-rs /usr/local/bin/
#+end_src

** Building from source

Install Rust using [[https://rustup.rs][rustup]]:

#+begin_src sh
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source ~/.cargo/env
#+end_src

Install the ALSA development library and build:

#+begin_src sh
sudo apt install libasound2-dev
cargo build --release
#+end_src

The binary will be at =target/release/udpiq-rs=. Copy it to your PATH:

#+begin_src sh
sudo cp target/release/udpiq-rs /usr/local/bin/
#+end_src

** ALSA configuration

The ALSA device must provide 2-channel (IQ) float samples at 78125 Hz.
If your hardware uses a different native format (e.g. =S32_LE=), use an
ALSA plug device in =~/.asoundrc= to convert:

#+begin_src conf
pcm.sdr {
    type plug
    slave {
        pcm "hw:vfzfpga,0"
        format S32_LE
    }
}
#+end_src

** Usage

#+begin_src sh
udpiq-rs stream <DEVICE> <ADDRESS> [OPTIONS]
#+end_src

| Argument    | Description              | Default |
|-------------+--------------------------+---------|
| =DEVICE=    | ALSA device name         |         |
| =ADDRESS=   | Destination IP address   |         |
| =-p, --port=  | Destination UDP port   |    7373 |
| =-n, --frames= | ALSA period size (frames) | 1024 |

Example:

#+begin_src sh
udpiq-rs stream sdr 192.168.1.100
#+end_src

** GNU Radio

Use a UDP Source block with:

- Port: 7373
- Payload size: 1472
- Type: float
- Vector length: 1

** Limitations

- Sample rate is hardcoded to 78125 Hz (VFZSDR FPGA clock).
- ALSA format is hardcoded to 2-channel interleaved float.
- IPv4 only.
- UDP is unreliable -- packets may be lost or reordered. Works well
  over LAN but will degrade over lossy links.
- No packet sequencing or timestamps -- the receiver cannot detect
  dropped packets.
- Non-blocking sends silently drop packets when the socket buffer is
  full.
